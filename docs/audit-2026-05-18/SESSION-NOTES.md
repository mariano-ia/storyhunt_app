Single-day record of what happened during the pre-OTA audit + lockdown sprint.
Written for future-you (or future-me) trying to understand why things look the

## What triggered the session

Mariano received the weekly conversion-review email at 1 PM. Numbers looked
great on the surface (37 purchases, $4.68 CAC, 8x improvement in view-to-purchase)
but a few things didn't add up:

- His Stripe balance only showed **one** $9.99 sale, not 37.
- The dashboard had sessions of advanced progress (26%, 64%, 97%) with no

- Hern├ín (the only "no" to the NYC gate) was unclear whether he had actually
  played or not.

He asked to dig into the data.

## What we found (early afternoon)

A scripted Firestore audit (`scripts/audit-week.js`, `audit-corrected.js`)
revealed:

1. The "37 purchases" was wrong. Real data: **1 paid sale ($9.99)** plus
   **17 free claims via STORYHUNT coupon**. PostHog was counting events with
   a `purchase` name regardless of `amount: 0`, AND likely double-firing
   because the Stripe webhook had no idempotency.
2. **TripAdvisor approval + OTA campaign launch was about to happen.** Mariano
   said: "I need a deep audit before that traffic hits us."
3. Spawned 4 parallel agents to cover security, funnel/UX, operational
   readiness, data integrity. Got ~130 findings back.

## What the agents found (the big ones)

- **Firestore rules had `allow read, write: if true` on `access_tokens`.**
  PoC'd it: dumped 59 customer emails via plain `curl` against the public
  Firestore REST API. Same was true for `discount_coupons` (read), and the
  write side was open on `user_sessions`, `sales`, `interactions`.
- `/api/dashboard/funnel` and `/api/campaigns` were both **unauth**. PoC'd:
  pulled revenue, masked emails, and $1,027.50 Meta spend without
  authentication.
- `verifyAuth()` only checked that a Firebase JWT was valid ÔÇö **not** that the
  caller was in the admins allowlist. Since Firebase email/password sign-up is
  on by default, anyone could mint themselves a user and call admin endpoints.
- Stripe webhook had **no idempotency**. On retry, it created duplicate
  tokens, sales, emails, coupon increments. This was the likely root cause of
  PostHog 37-vs-1.
- Tokens were `SH-XXXXXX` (6 chars from a 32-char alphabet, generated by
  `Math.random()`). ~30 bits of entropy, brute-forceable at the API level.
  Moot in practice because the Firestore leak dumped them directly, but worth
  fixing anyway.
- `llm_api_key` field on experiences leaked to the browser via public read on
  `experiences`.
- 33 sessions in April with no email attached ÔÇö closure-bug era. Already
  identified in memory but never cleaned up.
- No privacy policy / terms / refund policy pages. Stripe ToS + TripAdvisor
  listing require them.
- DNS apex `storyhunt.city` had no SPF record (only `send.storyhunt.city`
  had one).
- ~22 `/api/debug/*` routes still in production from various one-shot
  migrations. All gated by `CRON_SECRET` in query string.

Full report: `docs/audit-2026-05-18/AUDIT-REPORT.md`.

## What we shipped (afternoon Ôćĺ evening)

Single commit `b2974d6` to ABM main + a few followups + commits on StoryHuntWeb.

### Security perimeter
- Rewrote `firestore.rules`: closed reads + writes on PII collections. Server
  uses Admin SDK; client SDK can only read auth-required collections (sales,
  sessions, etc.) and the public-readable `experiences`.
- **Deployed rules to prod via `scripts/deploy-firestore-rules.js`** ÔÇö used
  the Firebase Rules REST API + the service account already in `.env.local`.
- `verifyAuth()` now requires the caller's email to be in `admins`
  collection. `isAdmin()` exposed for explicit checks.
- Bootstrapped `marianonoceti@gmail.com` into admins so I didn't lock Mariano
  out (`scripts/add-admin.js`).
- `/api/users/invite` migrated from client SDK to Admin SDK.
- `/api/dashboard/funnel` and `/api/campaigns` now require `verifyAuth`.
- Player reads of experiences now strip `llm_api_key` via `stripSensitive()`
  in `lib/firestore.ts`. Server reads still get the full doc.

### Stripe webhook
- Idempotency: `stripe_events/{event.id}` via `.create()`. Throws on duplicate,
  treated as "skip". Pattern: every webhook event has a Firestore doc keyed
  by `event.id`.
- Belt-and-suspenders: even after the event dedup, skip if a `sale` row exists
  for the same `stripe_session_id`.
- Tokens: `crypto.randomBytes(8)` mapped to a 32-char alphabet. 40 bits.
- Always return 200 once signature verifies. Errors are logged; we don't ask
  Stripe to retry a half-processed event.
- New event handlers: `charge.refunded`, `charge.dispute.created` (both
  revoke the access token + flag the sale as `status: 'refunded'`),
  `checkout.session.async_payment_succeeded`.
- Mark contact as `converted: true` on purchase, so nurturing cron stops
  emailing buyers.
- Forward `fbp`/`fbc`/`utm_*`/`client_ip`/`client_ua` from Stripe metadata
  into the CAPI Purchase event for proper Meta dedup.
- Mask email + token in console logs.

### Access tokens
- `/api/access/use` uses `FieldValue.increment(1)` (atomic).
- `/api/access/verify` regenerates tokens via `crypto.randomBytes`.
- `/api/sessions/find` accepts `access_token` as a fallback when email is
  missing ÔÇö recovers the orphan-session pattern from 2026-05-12.
- `?from=N` in the player no longer bypasses the NYC gate. Pre-fix, anyone
  with a token could craft `/play/[id]?token=X&from=1` to skip it.

### Checkout
- `KILL_SWITCH_PAYMENTS=true` env returns 503 on `/api/checkout`. Emergency
  stop without redeploy.
- Body length caps on `email`, `experience_id`, `coupon_code`.
- Captures `utm_source`/`utm_medium`/`utm_campaign`/`fbp`/`fbc`/`ga_client_id`/
  `referrer` from the client + `client_ip`/`client_ua` from headers, all
  forwarded to Stripe metadata.
- Coupon `valid_until` is now enforced server-side.

### Email hygiene
- `replyTo: 'hello@storyhunt.city'` on every Resend send (8 sites).
- `List-Unsubscribe` + `List-Unsubscribe-Post: One-Click` headers.
- New `/api/unsubscribe` endpoint + `/unsubscribe` page.
- Nurturing cron respects `unsubscribed: true` and skips `refunded` tokens.

### Cron + data integrity
- New `/api/cron/abandon-stale-sessions` (daily 15:00 NYC) flips `in_progress`
  sessions older than 12h to `abandoned`.
- `/api/cron/post-experience-email` now filters by `first_used_at` (not
  `created_at`) ÔÇö fixes the bug where tourists buying ahead never got a
  review email.
- Marks `review_email_sent: true` BEFORE the actual send (avoids resend on
  mid-flight crash). Tracks `review_email_attempts` + `review_email_failed`.
- New `cron_runs` collection for cron-execution logging.

### XSS / hardening
- `renderMessage` only renders `http(s)`, `mailto:`, `tel:`, `/` URLs.
  `javascript:` and `data:` URLs from LLM/user content are downgraded to plain
  text.
- Security headers in `next.config.ts`: X-Frame-Options DENY, nosniff,
  Referrer-Policy strict-origin, Permissions-Policy, HSTS preload, X-Robots-Tag
  noindex on `/play/*` + `/dashboard/*`.

### Player UX
- Default lang = `'en'` (was `'es'`) ÔÇö tourist traffic via OTA is English-first.
  Token-stored lang overrides via query param.
- Paywall blocked screen is localized and includes a `mailto:` support contact.
- Chat header has a `?` help icon Ôćĺ `mailto:hello@storyhunt.city`.
- Chat timestamp uses America/New_York TZ (was hardcoded `es-AR`).

### Dashboard
- `/dashboard/sessions` adds NYC? column with the raw `in_nyc_reply` on hover.

### Legal pages (StoryHuntWeb)
- `/privacy`, `/terms`, `/refund-policy` static HTML pages added.
- Footer of `index.html` linked to all four.
- `cleanUrls: true` added to `vercel.json` so paths resolve without `.html`.

### DNS
- Apex SPF `v=spf1 include:amazonses.com ~all` (Mariano added manually in
  Squarespace Domains; verified via `dig`).

### Cleanup
- 33 legacy April user_sessions (no-email, closure-bug era) deleted.

## How to reproduce the audit if you doubt the rules are deployed

```bash

curl "https://firestore.googleapis.com/v1/projects/storyhunt-platform-961ec/databases/(default)/documents/discount_coupons?pageSize=1"

# Should be 401:
curl https://storyhunt-app.vercel.app/api/dashboard/funnel?days=7
curl https://storyhunt-app.vercel.app/api/campaigns

# Should be 200 (player needs it):
curl "https://firestore.googleapis.com/v1/projects/storyhunt-platform-961ec/databases/(default)/documents/experiences?pageSize=1"
```

Or just `bash scripts/post-deploy-smoke.sh` ÔÇö runs all 32.
